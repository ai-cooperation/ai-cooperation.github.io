Skill: Presentation Pipeline (PPTX)

Overview

Fully automated Quarto-to-PPTX presentation pipeline. From structured specs to professionally styled PowerPoint, entirely driven by AI agents.

Module ID skill-presentation-pptx
Version 2.0.0
Spec v1.0
License Apache 2.0
GitHub ai-cooperation/skill-presentation-pptx

Quick Start

git clone https://github.com/ai-cooperation/skill-presentation-pptx
cd skill-presentation-pptx

# Install dependencies
pip install python-pptx lxml
# Requires: Quarto CLI

# Compile an example
bash tools/compile_pptx.sh examples/deck-basic.qmd basic outputs deck.pptx

Pipeline Architecture

User Request
    ↓
[Planner Skill] → deck_spec.json
    ↓                (template_style + layout_plan)
[Worker Skill]  → deck.qmd
    ↓                (correct QMD syntax per layout)
[compile_pptx.sh]
    ├─ Phase 1: build_template.py → template-{style}.pptx
    ├─ Phase 2: quarto render     → raw PPTX
    └─ Phase 3: postprocess.py    → final PPTX (patched)
    ↓
Final Styled PPTX

7 Supported Layouts

The module’s reference knowledge base teaches AI agents how to trigger each PowerPoint layout through specific QMD syntax:

Layout QMD Trigger
Title Slide YAML front matter title/subtitle
Title and Content ## Heading + bullets
Two Content ## Heading + :::: {.columns} (text-only)
Comparison ## Heading + :::: {.columns} (text + image)
Content with Caption ## Heading + text + image (no columns)
Section Header # Heading (level-1)
Background Image ## {background-image="..."} + &nbsp;

Key Technical Innovations

  1. Layout Trigger Rules — Documented how QMD structure maps to Pandoc layout selection (undocumented by Pandoc)
  2. Dynamic Template Building — No manual PowerPoint editing; templates built programmatically with python-pptx
  3. Postprocessing — Fixes properties Pandoc strips (vertical centering, etc.)
  4. AI Knowledge Base — Reference docs designed for AI consumption, not human tutorials
